# تطبيق بوابة الرسائل (Messaging Gateway App)

## نظرة عامة
تطبيق بوابة الرسائل هو تطبيق أندرويد مصمم ليعمل كجسر بين أنظمة العملاء الخارجية وشبكات المراسلة (واتساب و SMS). يعتمد التطبيق على نموذج **"جهازك هو البوابة" (BYOD)**، حيث يتم تثبيت التطبيق على جهاز أندرويد حقيقي ليحوله إلى بوابة إرسال.

## الميزات الرئيسية
-   **بوابة واتساب**: يستخدم متصفح داخلي (WebView) لتشغيل "واتساب ويب" وأتمتة عملية الإرسال عبر حقن أكواد برمجية (DOM Manipulation).
-   **بوابة SMS**: يستخدم إمكانيات الجهاز الأصلية لإرسال الرسائل النصية.
-   **اتصال مزدوج**:
    -   يتصل بـ **سيرفر "آيديا" (Idea Backend)** لإدارة الاشتراكات والمصادقة.
    -   يتصل بـ **سيرفر العميل الخاص** عبر تقنية Socket.io لاستقبال أوامر الإرسال.

## التقنيات المستخدمة (Technology Stack)

### تطبيق الأندرويد (البوابة)
-   **بيئة التشغيل**: React Native (Expo SDK 54).
-   **اللغة**: TypeScript.
-   **التوجيه (Routing)**: Expo Router.
-   **واجهة المستخدم**: NativeWind (Tailwind CSS).
-   **الاتصال اللحظي**: `socket.io-client` (للربط مع سيرفرات العملاء).
-   **أتمتة واتساب**: `react-native-webview` مع حقن JavaScript.
-   **التخزين المحلي**: `AsyncStorage` و `expo-secure-store`.
-   **عميل API**: مكتبة tRPC (للتواصل مع سيرفر آيديا).

### سيرفر آيديا (Backend)
-   **بيئة التشغيل**: Node.js.
-   **إطار العمل**: Express.js.
-   **واجهة البرمجة**: tRPC.
-   **قاعدة البيانات**: MySQL (باستخدام Drizzle ORM).
-   **المصادقة**: JWT.

### لوحة التحكم الإدارية (Admin Dashboard)
-   **إطار العمل**: React (Vite).
-   **إدارة الحالة**: Zustand.
-   **التنسيق**: Tailwind CSS.

## الإعداد والتثبيت
1.  **تثبيت المكتبات**:
    ```bash
    pnpm install
    ```
2.  **تشغيل سيرفر التطوير**:
    ```bash
    pnpm dev
    ```
3.  **تشغيل تطبيق الأندرويد**:
    ```bash
    pnpm android
    ```

## مفهوم الاستخدام
1.  يقوم المستخدم بتثبيت التطبيق وتسجيل الدخول (مصادقة مع سيرفر آيديا).
2.  يدخل المستخدم **رابط Socket.io الخاص به** في شاشة "إدارة الاتصال" (Connection Manager).
3.  يتصل التطبيق بسيرفر العميل ويستمع لأوامر `send_message`.
4.  يقوم المستخدم بتسجيل الدخول لـ واتساب ويب داخل التطبيق.
5.  عند وصول أمر إرسال، ينفذ التطبيق المنطق المطلوب عبر WebView (للواتساب) أو النظام الأصلي (للرسائل النصية).
